MePersonality presents a distributed multi-agent collaborative personalisation platform. Since the main purpose is to enable personalisation of browsed web content to user, MePersonality provides an interface for user interests model access. Via personalisation extensions, user can extend this model even further and based on gathered information about user interests and behaviour, user can modify content of browsed web pages to work more efficiently and comfortably.

Personalisation extensions

Personalisation extensions are basically pieces of JavaScript code which can personalise the web browsing according to user needs. Besides basic functionality of JavaScript language, MePersonality provides wider possibilities within personalisation extensions including:
  • Built-in support of jQuery framework
  • Use of other external JavaScript files, e.g. another third-party library
  • Cross-origin requests
  • Access to user browsing history
  • Persistent storage via database API
  • Personalisation API
  • Communication API
This functionality is contained in a simple and fully asynchronous API which contributes to better user experience and more comfortable development of high-quality personalisation extensions. Moreover, this platform is aimed to be independent of contained web browser, so that developer does not need to concern about any compatibility issues and is able to focus on the main work goal - the personalisation.

Database API

Database API provides a unified interface to access the database (one of the advantages of our solution), so that developers need not worry about what type of database is currently implemented in the browser. Each extension has its own database, so different extensions do not limit each other. The database API currently exposes the following interface:

Method Description
clear() Deletes all records from the database.
get() Returns a record with specified key.
set() Sets the value of the record with specified key to a given value. This value can be of any JavaScript type that can be converted to a JSON string. If the key is already present in the database before insertion, its value is updated, otherwise the key is created and its value is adjusted according to the value specified as a parameter to this function.
remove() Removes record with the given key from the database. If no such key exists in the database, nothing changes.

Personalisation API

Personalization API enables access to all user information we have obtained within the user interest modelling process. It provides the following functionality:

Method Description
getUrlsByTag() Returns URL list sorted by the relevancy of given tag.
getTagsByUrl() Returns weighted tag list for given URL address.
getDomains() Returns domain list sorted by the visit count.
getUserTags() Returns weighted list of tags with the highest relevancy within the whole user model (global user interests).
getUserTagsByDomain() Returns weighted list of tags with the highest relevancy within the given domain (local user interests).
getTagRelevancy() Returns relevancy of given tag.
getUrlTitle() Returns title for given URL.
getVisitCount() Returns visit count of given URL.
